# README-backend.md
## Project Description
The CodePal API is a RESTful API that provides a comprehensive set of endpoints for interacting with the CodePal system. The API is designed to be used by the frontend application, as well as by third-party developers who wish to integrate CodePal into their own applications.

## Tech Stack Used
* Python as the backend language
* FastAPI as the web framework
* MongoDB as the database
* JWT (JSON Web Tokens) for authentication

## Setup Instructions
To set up the CodePal API backend, follow these steps:

1. **Clone the repository**: Clone the CodePal API repository from GitHub using `git clone https://github.com/your-username/codepal-api.git`.
2. **Install dependencies**: Install the required dependencies using `pip install -r requirements.txt`.
3. **Create a MongoDB database**: Create a new MongoDB database and add a user with read and write permissions.
4. **Create environment variables**: Create a new file named `.env` in the root directory and add the following environment variables:
	* `DB_URI`: the MongoDB database URI
	* `JWT_SECRET`: the secret key for signing JWT tokens
	* `PORT`: the port number to run the API on (default is 3000)

## Environment Variables
The following environment variables are required:

* **DB_URI**: the MongoDB database URI (e.g. `mongodb://localhost:27017/codepal`)
* **JWT_SECRET**: the secret key for signing JWT tokens (e.g. `secretkey123`)
* **PORT**: the port number to run the API on (e.g. `3000`)

## API Endpoints Overview
The CodePal API has the following endpoints:

* **POST /api/v1/auth/login**: logs in a user and returns a JWT token
* **GET /api/v1/projects**: gets a list of projects for the authenticated user

## How to Run Locally
To run the CodePal API locally, follow these steps:

1. **Start the MongoDB database**: start the MongoDB database using `mongod` or `sudo service mongod start`.
2. **Start the API**: start the API using `uvicorn main:app --host 0.0.0.0 --port 3000`.
3. **Access the API**: access the API using `http://localhost:3000/api/v1` or the port number specified in the `.env` file.

## How to Run Tests
To run the tests, follow these steps:

1. **Install Pytest**: install Pytest using `pip install pytest`.
2. **Run the tests**: run the tests using `pytest`.
3. **View test results**: view the test results in the terminal or in a test report generated by Pytest.
